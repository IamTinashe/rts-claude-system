# Report Generation Skill

## Purpose

Produce McKinsey-grade RTS deliverables that are clear, rigorous, and actionable.
Every output should be indistinguishable from what a top-tier consulting firm would deliver.

## Writing Standards

### Tone
- Authoritative but not arrogant
- Direct — lead with the conclusion, then support it
- Precise — specific numbers, dates, and actions rather than vague qualifiers
- Balanced — acknowledge risks and uncertainties explicitly

### Structure (Pyramid Principle)
1. Start with the answer/recommendation
2. Support with 3-5 key findings
3. Back each finding with data and analysis
4. Provide detailed appendices for those who want to drill down

### Formatting Rules
- Executive Summary: max 2 pages, standalone readable
- Findings: one key message per section, supported by data
- Recommendations: specific, actionable, with owner + timeline
- Every chart needs a title that states the insight, not just the data
  - BAD: "Revenue by Quarter"
  - GOOD: "Revenue declined 15% YoY driven by Client A contract loss"

## Report Template

### 1. Executive Summary (2 pages)
- Engagement context (1 paragraph)
- Key findings (3-5 bullets, each with a number)
- Transformation opportunity summary (total $ value)
- Critical risks (2-3 bullets)
- Recommended immediate actions (30-day priorities)

### 2. Situation Overview (3-5 pages)
- Company background
- Distress trigger and timeline
- Stakeholder landscape
- Scope of this engagement

### 3. Financial Analysis (8-12 pages)
- Adjusted EBITDA bridge (with waterfall chart)
- Revenue and margin trends
- Working capital analysis
- Debt structure and covenant status
- Liquidity assessment
- Cash flow projections (13-week and 3-year)

### 4. Operational Assessment (8-12 pages)
- Assessment by lever (revenue, cost, working capital, assets, org, tech, strategy)
- For each lever: current state, gap, opportunity, implementation path
- Benchmarking where data available

### 5. Transformation Roadmap (5-8 pages)
- Value bridge: current → transformed EBITDA
- Initiative prioritization matrix (impact × feasibility)
- 30/60/90-day implementation plan
- Resource requirements
- Key milestones and decision points

### 6. Risk Factors (2-3 pages)
- Execution risks by initiative
- External risks (market, regulatory, competitive)
- Stakeholder risks (management capability, creditor alignment)
- Mitigants for each risk

### 7. Appendices
- Detailed financial model
- Data sources and confidence levels
- Full data request list with status
- Glossary of terms
- Team and contact information

## Quality Checklist (Pre-Delivery)

Before finalizing any deliverable:

- [ ] Every number traces to a source document
- [ ] All assumptions are explicitly stated
- [ ] Confidence levels assigned to all findings
- [ ] Executive summary is readable standalone
- [ ] Recommendations are specific and actionable (who, what, when)
- [ ] Charts have insight-driven titles
- [ ] No unsupported claims or vague language
- [ ] Spell-check and formatting consistent throughout
- [ ] Page numbers, headers, footers correct
- [ ] File named according to convention: `[Client]_RTS_Report_[Date]_v[#].pdf`
